# Руководство по работе с RichText в проекте

## Проблема
При редактировании RichText полей в Strapi (отступы, жирный текст, списки и т.д.), React отображал их как обычный текст без форматирования. Strapi возвращает RichText в формате Markdown (например, `**жирный текст**`), а не HTML.

## Решение
Создан компонент `MarkdownRenderer` для парсинга Markdown-контента из Strapi и обновлены все компоненты для его использования.

## Обновленные компоненты

### 1. MarkdownRenderer.jsx (новый компонент)
- Парсит Markdown-контент в HTML
- Поддерживает: заголовки, жирный/курсивный текст, списки, ссылки, зачеркнутый текст
- Безопасно рендерит HTML через `dangerouslySetInnerHTML`

### 2. SingleArtist.jsx
- Поле `bio` теперь отображается с Markdown-форматированием
- Использует `MarkdownRenderer` компонент

### 3. About.jsx
- Поля `textP1` и `textP2` отображаются с Markdown-форматированием
- Сохранены CSS классы для стилизации

### 4. News компоненты
- `SingleNews.jsx` - полное описание новости
- `NewsItem.jsx` - краткое описание в списке
- Поле `text` отображается с Markdown-форматированием

### 5. Performance компоненты
- `SinglePerformance.jsx` - полное описание спектакля
- `Performance.jsx` - краткое описание в списке
- Поле `description` отображается с Markdown-форматированием

### 6. SingleTeam.jsx
- Поле `bio` отображается с Markdown-форматированием

## CSS стили
Создан файл `src/styles/components/richtext.css` с полным набором стилей для:
- Заголовков (h1-h6)
- Параграфов
- Списков (ul, ol)
- Жирного и курсивного текста
- Ссылок
- Цитат
- Кода
- Изображений
- Таблиц
- Адаптивности для мобильных устройств

## Безопасность
⚠️ **Важно**: Использование `dangerouslySetInnerHTML` может быть небезопасным, если контент не проверен. В данном случае контент приходит из Strapi CMS, который контролируется администратором, поэтому риск минимален.

## Как использовать
1. В Strapi админке редактируйте RichText поля с форматированием
2. Сохраняйте изменения
3. В React приложении форматирование будет отображаться корректно

## Поддерживаемые элементы форматирования Markdown
- **Жирный текст** (`**текст**` или `__текст__`)
- *Курсив* (`*текст*` или `_текст_`)
- ~~Зачеркнутый текст~~ (`~~текст~~`)
- Заголовки (`# Заголовок 1`, `## Заголовок 2`, `### Заголовок 3`)
- Списки (`- элемент` или `* элемент` или `1. элемент`)
- Ссылки (`[текст](url)`)
- Переносы строк (двойной перенос для параграфов)

## Тестирование
Для проверки работы:
1. Откройте Strapi админку
2. Отредактируйте любое RichText поле (например, bio артиста)
3. Добавьте Markdown форматирование:
   - `**Жирный текст**` для жирного текста
   - `*Курсив*` для курсива
   - `# Заголовок` для заголовков
   - `- Элемент списка` для списков
4. Сохраните изменения
5. Обновите React приложение
6. Проверьте, что форматирование отображается корректно

## Пример использования в Strapi
```
# Биография артиста

Это **жирный текст** и *курсивный текст*.

## Достижения
- Первая роль в 2010 году
- Лауреат премии в 2015 году
- **Главная роль** в спектакле "Гамлет"

Подробнее на [сайте театра](https://example.com)
```
